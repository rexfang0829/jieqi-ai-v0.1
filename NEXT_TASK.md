# Next Task

## 下一輪目標

建議下一輪繼續 Phase 1，做「局面儲存 / 載入」的最小版本。

## 為什麼要做

目前已經可以清空棋盤並手動建立局面，但手動輸入一盤棋很花時間。如果不能儲存，重新整理頁面或重開專案後就要重建一次。先做簡單儲存/載入，可以讓實戰手動輸入流程更穩。

## 具體要改哪些功能

建議範圍：

- 在 UI 加上「儲存目前局面」按鈕。
- 在 UI 加上「載入已儲存局面」按鈕。
- 儲存內容包含：
  - 棋盤
  - 輪到哪方
  - status
  - history 可先不做，或只保留空陣列
- 儲存方式可以先用 `localStorage`。
- 新增測試確認儲存與載入資料格式。

## 不要做哪些超出範圍的事

本任務不要做：

- Belief State
- Monte Carlo
- OCR
- Ponder
- 自動截圖辨識
- 大型重構
- 雲端同步
- 多局面管理

## 完成標準

- 可以把目前手動編輯出的棋盤存起來。
- 可以從瀏覽器本機載入已儲存棋盤。
- 載入後仍可繼續編輯或走棋。
- `npm test` 通過。
- `npm run build` 通過。
- 更新 `CODEX_STATUS.md` 與 `NEXT_TASK.md`。
- 測試與 build 通過後 commit 並 push。
